# Process

## Подход
Работал с Claude как с напарником, а не как с генератором кода.

## Этапы

### 1. Анализ ТЗ
Скопировал полное ТЗ в Claude. Совместно разобрали требования и выделили три независимых флоу: `login`, `register`, `recover`.

### 2. Архитектура
До написания кода обсудили расслоение ответственностей:
- `routes/` — защита и навигация
- `modules/auth/` — формы и схемы
- `services/` — HTTP-слой
- `lib/auth/` — контекст и хранилище токенов

Инструменты выбирал самостоятельно, а Claude помогал аргументировать trade-offs.

### 3. Выбор бэкенда
Форк бэкенда выбирал из нескольких репозиториев других кандидатов. Первый вариант не подошел (gRPC без адаптера на фронте). Остановился на третьем: Go + GraphQL, минимальные зависимости.

Запустил локально через Docker (`make up`), подключил фронт и работал с живым API.

### 4. HTTP-клиент
Конфигурацию `ky` адаптировал из собственного опыта: добавил хуки для перехвата `401` и автоматического refresh токена. Claude помог подстроить реализацию под контекст задачи.

### 5. Написание кода
Весь код проходил через мое понимание и ревью. Часть писал вручную, часть адаптировал из предложений Claude; в обоих случаях понимал каждую строку до попадания в репозиторий.
